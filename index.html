🌍 MASALAH NYATA: Mengukur Lahan Pertanian

Pak Tani memiliki lahan berbentuk tidak beraturan di pinggir sungai. Dengan alat ukur GPS, ia mendapatkan 8 titik koordinat batas lahan. Bagaimana menghitung luasnya secara akurat?

Tanpa kalkulus: harus bagi jadi segitiga-segitiga (tidak akurat).
Dengan kalkulus: integral membagi lahan menjadi irisan tipis!

📐 KONSEP DASAR INTEGRAL

Rumus Dasar: Luas = ∫ f(x) dx

"Membagi area menjadi irisan-irisan tipis, lalu menjumlahkan luasnya"

🔢 Langkah-langkah Menghitung Luas dengan Integral:

  1. Bagi daerah lahan menjadi irisan-irisan vertikal dengan lebar Δx (sangat kecil)
  2. Setiap irisan dianggap sebagai persegi panjang dengan tinggi f(x)
  3. Luas satu irisan ≈ f(x) × Δx
  4. Jumlahkan semua irisan: Σ f(x) × Δx
  5. Jika Δx → 0 (mendekati nol), jumlah Riemann menjadi integral tentu: ∫ f(x) dx

🧮 4 METODE PERHITUNGAN LUAS

📐 1. Poligon (Shoelace) - Referensi Eksak

📌 Rumus: ½|Σ(xᵢyᵢ₊₁ - xᵢ₊₁yᵢ)| → hasil geometri tepat (bukan aproksimasi)

🧱 2. Riemann - 3 Varian

📌 Riemann Kiri: ambil tinggi di ujung kiri → underestimate (kurang)

📌 Riemann Kanan: ambil tinggi di ujung kanan → overestimate (lebih)

📌 Riemann Tengah: ambil tinggi di titik tengah → paling akurat

📊 3. Trapezoida - 2 Varian

📌 Uniform: partisi sama rata, cepat dan sederhana

📌 Adaptive: partisi lebih rapat di area melengkung, lebih akurat

📈 4. Simpson - 3 Varian

📌 Simpson 1/3: aproksimasi parabola, partisi genap

📌 Simpson 3/8: aproksimasi kubik, partisi kelipatan 3

📌 Auto: pilih metode terbaik otomatis

🔍 PERAN INTEGRAL DALAM PROGRAM:

  • Irisan (Partisi): Lahan dibagi menjadi n bagian vertikal. Nilai n diatur melalui fitur Partisi (100, 200, 500, 1000).
  • Lebar irisan (Δx): Δx = (xₘₐₖₛ - xₘᵢₙ) / n → semakin besar n, semakin kecil Δx, hasil semakin akurat.
  • Tinggi irisan (f(x)): Ditentukan dari titik potong garis vertikal dengan batas lahan. Bergantung pada metode yang dipilih:
    • Riemann Kiri: f(x) diambil di ujung kiri interval
    • Riemann Kanan: f(x) diambil di ujung kanan interval
    • Riemann Tengah: f(x) diambil di titik tengah interval
    • Trapezoida: f(x) diambil di kedua ujung, lalu dirata-rata
    • Simpson: f(x) diambil dengan bobot parabola/kubik
  • Jumlah Riemann: Σ (tinggi × lebar) → aproksimasi luas total lahan.
  • Limit n → ∞: Jika Δx → 0 (n sangat besar), jumlah Riemann mendekati nilai eksak: ∫ f(x) dx.
  • Implementasi dalam Program:
    • Pengguna memasukkan titik koordinat lahan
    • Program membagi lahan menjadi n irisan vertikal
    • Menghitung luas dengan 4 metode (Poligon, Riemann, Trapezoida, Simpson)
    • Membandingkan error terhadap metode Poligon (referensi eksak)
📚 PENJELASAN METODE & MATERI KULIAH (BAB 5 & BAB 7) ▶

📐 BAB 5: APLIKASI INTEGRAL

Dalam program ini, integral digunakan untuk menghitung luas lahan dengan konsep:

  • Luas = ∫ f(x) dx - Membagi area menjadi irisan tipis
  • Setiap irisan memiliki luas = tinggi × lebar (Δx)
  • Menjumlahkan semua irisan = integral tentu
  • Semakin kecil Δx (partisi banyak), semakin akurat hasilnya

🧮 BAB 7: TEKNIK INTEGRASI NUMERIK

📐 1. Metode Poligon (Shoelace) - Referensi

Rumus: Luas = ½|Σ(xᵢyᵢ₊₁ - xᵢ₊₁yᵢ)|

Menghitung luas secara geometris dari koordinat titik. Metode eksak, digunakan sebagai referensi kebenaran.

🧱 2. Metode Riemann

Rumus: Luas ≈ Σ f(x) · Δx

Membagi area menjadi persegi panjang. Tersedia 3 varian:

  • Riemann Kiri: Mengambil tinggi dari titik kiri interval → underestimate
  • Riemann Kanan: Mengambil tinggi dari titik kanan interval → overestimate
  • Riemann Tengah: Mengambil tinggi dari titik tengah interval → paling akurat

Error ~0.5-2% tergantung bentuk lahan.

📊 3. Metode Trapezoida

Rumus: Luas ≈ Σ (f(xᵢ) + f(xᵢ₊₁))/2 · Δx

Menghubungkan titik dengan garis lurus membentuk trapesium. Tersedia 2 varian:

  • Uniform: Partisi sama rata, cepat dan sederhana
  • Adaptive: Partisi lebih rapat di area melengkung, lebih akurat

Error ~0.25-1% tergantung bentuk lahan.

📈 4. Metode Simpson

Mengaproksimasi dengan parabola/kubik. Tersedia 3 varian:

  • Simpson 1/3: Aproksimasi parabola, partisi genap
  • Simpson 3/8: Aproksimasi kubik, partisi kelipatan 3
  • Auto: Pilih metode terbaik otomatis

Error ~0.05-0.5% untuk bentuk mulus, metode paling akurat.

💡 CATATAN: Trapezoida Adaptive

Metode Adaptive secara otomatis membagi interval lebih rapat di area yang melengkung. Hasilnya: akurasi lebih tinggi dengan jumlah partisi yang sama dibandingkan metode Uniform.

💡 CATATAN: Simpson Auto

Mode Auto akan memilih Simpson 1/3 jika partisi genap, dan Simpson 3/8 jika partisi kelipatan 3.
